《数据结构与算法》笔记与杂谈

绪论

计算

  • 什么是算法？计算即信息处理，是借助某种工具，遵照一定规则，以明确而机械的形式执行的操作。计算模型，即计算机，是信息处理工具。算法应有输入输出，正确性，确定性，可行性与有穷性(Hailstone Sequence)。

  • 什么是好算法？我们在意的是，效率。

Algorithms + Data Structures = Programs

(Algorithms + Data Structures) x Efficiency= Computation

计算模型

  • 效率的测度：这里关注时间成本
  • $T_A(P)$ := 算法A求解问题实例 P 的计算成本？X
    • 划分等价类：根据问题实例的规模
    • $T_A(n)$​ 用算法 A 求解某一问题规模为 n 的实例，所需的计算成本 √
    • 记作 $T(n)$​, and defined by the time cost of the worst case.
  • 图灵机模型
    • Components

        • L/R: move head

渐进复杂度

  • 计算成本如何随着问题规模的增长而增长，关心足够大的问题
    • 这里为什么一般考虑时间复杂度？就渐进复杂度的意义而言，在任一算
      法的任何一次运行过程中所消耗的存储空间，都不会多于其间所执行基本操作的累计次数。时间复杂度本身就是空间复杂度的一个天然的上界。
  • Big-$O$ Notation
    • $T(n) = O( f(n) ) = \Omega (f(n)) = \Theta(f(n))…$​
    • 高效解 $O(1), O(lg(n))$
    • 有效解 $O(n^c)$
    • 难解 $O(2^n)$​
    • Example: 2-subset
      • 2-Subset is NP-complete
      • 目前的计算模型不存在在多项式时间内回答此问题的解法

    • 递归：递归跟踪+递推方程
    • 猜测+验证
  • 级数
    • 算术级数 $T(n) = 1+2+…+n = O(n^2)$
    • 幂方级数 $\sum_{k=0}^{n}k^d = O(n^{d+1})$
    • 几何级数 $\sum_{k=0}^{n}a^k = O(a^k)$
    • 收敛级数 $O(1)$
    • 调和级数 $\sum_{k=1}^{n}\frac 1 k = O(log(n))$
    • 对数级数 $\sum_{k=1}^{n}log \ k= O(nlog(n))$​
    • 循环…(TO ADD)
  • 冒泡排序作为例子
    • 复杂度分析

迭代和递归

  • Decrease and conquer
    • 将大规模问题划分为两个子问题，其一平凡，另一规模缩减
    • 数组倒置
  • 递归跟踪 Recursion trace
  • 递推方程 $T(n) = T(n-1) + O(1)$, where $T(0) = O(1)$
  • Divide and conquer
    • 将大规模问题划分成若干规模缩减的子问题，规模大体相当
  • Master Theorem (To Add…)
    • $T(n) = a*T(\frac n b) + O(f(n))$，取决于$a$, $b$, $f(n)$​.​
    • 我们通过比较 $n^{\log_ba}$ 与 $f(n)$ 的大小，做出以下总结
      • $f(n) = O(n^{log_ba-\epsilon})$, 则 $T(n) = \Theta(n^{log_ba})$
      • $f(n) = \Omega(n^{log_ba+\epsilon})$​, 则 $T(n) = \Theta(f(n))$​
      • $f(n)=\Theta(n^{log_b(a)}log^k(n))$​，则$T(n)=\Theta(n^{log_ba}log^{k+1}(n)$​​

位图(Bitmap)

  • 使用单个 bit 来作为某个元素是否存在于集合中的记录
  • 作用
    • 节省存储空间
    • 快速去重
    • 快速排序
  • 具体实现，使用位操作
#pragma once
class Bitmap
{
private:
    int _capacity; // [0, _capacity)
    char *_data;

public:
    Bitmap(int n)
    {
        this->_capacity = n;
        _data = new char[(n + 7) % 8];
    }
    ~Bitmap() { delete[] _data; }

    bool test(int n)
    {
        return _data[n >> 3] & (0x80 >> (n & 0x07));
    }

    void set(int n)
    {
        _data[n >> 3] |= (0x80 >> (n & 0x07));
    }

    void clear(int n)
    {
        _data[n >> 3] &= (~(0x80 >> (n & 0x07)));
    }
};
  • 快速初始化
    • 使用校验环
    • 以空间作为补偿，节省初始化时间
